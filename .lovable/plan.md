

# Draft User Breach Notification Email

## What This Does
Creates a professional, reassuring email notification to MobiRides users informing them of the security incident, what was potentially affected, what was NOT affected, and what steps they should take.

## Output
A markdown file at `docs/security/USER_BREACH_NOTIFICATION_EMAIL.md` containing the ready-to-send email text.

## Key Points to Include (from your briefing)
- **What happened**: A security incident was detected and neutralized
- **What was NOT compromised**: Storage/verification documents; passwords (stored hashed)
- **What may have been compromised**: Email addresses and phone numbers
- **User actions required**:
  1. Change MobiRides password (precautionary)
  2. Change email account password if same password was reused
  3. Report suspicious activity (calls from international numbers, suspicious emails)
- **Trusted domains**: Only `mobirides.com` and `mobirides.africa`
- **Contact**: WhatsApp +267 78 689 464
- **Regulatory**: Report being filed with the Data Protection Commission (IDPC)
- **Reassurance**: Breach neutralized, security improvements implemented
