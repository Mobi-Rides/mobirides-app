Based on the completed frontend and database work, here is the plan for **Phase 5: Edge Functions & Automation**. This phase implements the secure backend logic for processing payments and managing the booking lifecycle.

## Phase 5: Edge Functions (Backend Logic)

### 1. Payment Processing (`initiate-payment`)
*   **Purpose**: Securely create payment transactions and interface with payment providers (PayGate/OrangeMoney).
*   **Logic**:
    *   Validate booking eligibility (status, ownership).
    *   Calculate/Verify total amount from DB to prevent tampering.
    *   Insert `payment_transactions` record with `initiated` status.
    *   Return payment parameters or redirect URL to client.

### 2. Webhook Handling (`payment-webhook`)
*   **Purpose**: Handle asynchronous payment status updates from providers.
*   **Logic**:
    *   Receive provider payload.
    *   Update `payment_transactions` status (`completed` or `failed`).
    *   **On Success**:
        *   Update booking status to `confirmed`.
        *   Call `credit_pending_earnings` database function to log host earnings.
        *   Send confirmation notification (via `notifications` table).

### 3. Earnings Release (`release-earnings`)
*   **Type**: Scheduled Task (Cron).
*   **Purpose**: Automatically move funds from "Pending" to "Available" 24 hours after rental ends.
*   **Logic**:
    *   Query completed bookings where rental end time was > 24 hours ago.
    *   Call `release_pending_earnings` stored procedure for each.

### 4. Booking Expiration (`expire-bookings`)
*   **Type**: Scheduled Task (Cron).
*   **Purpose**: Cancel bookings that haven't been paid within the deadline.
*   **Logic**:
    *   Query bookings with status `awaiting_payment` where `payment_deadline` has passed.
    *   Update status to `expired`.

### 5. Withdrawal Processing (`process-withdrawal`)
*   **Purpose**: Handle host payout requests.
*   **Logic**:
    *   Validate host balance (double-check).
    *   Record withdrawal request.
    *   (Future) Trigger automated bank transfer/mobile money payout.

**Technical Stack**: Supabase Edge Functions (Deno/TypeScript).
