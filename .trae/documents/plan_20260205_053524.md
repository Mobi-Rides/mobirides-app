Based on the completion of the core payment flows, here is the plan for the final **Phase 6: Admin Tools & Insurance**. This phase ensures that the platform can manage insurance premiums and claims effectively, fulfilling the compliance and financial tracking requirements.

## Phase 6: Admin Tools & Insurance Implementation Plan

### 1. Insurance Remittance Dashboard (`AdminRemittanceDashboard.tsx`)
*   **Purpose**: Track insurance premiums collected from renters that need to be remitted to the insurance provider (Pay-U).
*   **Features**:
    *   **Metrics Cards**: Total premiums collected, pending remittance, total remitted.
    *   **Batch List**: View past remittance batches (`premium_remittance_batches`).
    *   **Pending Policies Table**: List of bookings/policies where premium is collected but not yet remitted.
    *   **Action**: "Create Remittance Batch" button to mark pending premiums as remitted and generate a batch record.

### 2. Claims Management Updates (`AdminClaimsDashboard.tsx`)
*   **Purpose**: Update the existing claims dashboard to handle external provider references and excess payments.
*   **Updates**:
    *   Add columns for `payu_claim_reference` and `external_status`.
    *   Add action to "Request Excess Payment" from renter (if liability applies).
    *   Add status tracking for "Excess Paid" vs "Excess Pending".

### 3. Excess Payment Flow (`ExcessPaymentModal.tsx`)
*   **Purpose**: Allow renters to pay their insurance excess/liability if a claim is processed against them.
*   **Features**:
    *   Modal triggered from Renter Dashboard if a claim requires excess payment.
    *   Reuses `useBookingPayment` hook but for `excess_payment` type (may need slight hook tweak or new param).
    *   Updates `insurance_claims` table upon success.

### 4. Navigation & Routing
*   **Admin Sidebar**: Add "Remittances" link under Finance/Insurance section.
*   **Routes**: Register new admin routes in `App.tsx`.

**Outcome**: Full financial loop closure—from renter payment -> host earnings -> insurance provider remittance.
