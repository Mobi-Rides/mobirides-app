## 🟢 Context & Objective
The MobiRides Android application (built with Capacitor) currently operates with "web-only" paradigms for notifications and maps. During the Android Gradle build process, a warning is issued indicating that `google-services.json` is missing. 

The objective is to integrate the **Google Services SDK** and **Firebase Cloud Messaging (FCM)** to enable native push notifications, which are critical for rental reminders, payment confirmations, and security alerts.

---

## 🛠️ Implementation Steps

### 1. External dependencies (Firebase Console)
- [ ] Create a Firebase project named `Mobi Rides`.
- [ ] Register the Android application with package name `com.mobirides.app`.
- [ ] Download `google-services.json` and upload/place in `./android/app/`.

### 2. Gradle & Native Configuration
- [ ] Verify `android/app/build.gradle` automatically applies the `com.google.gms.google-services` plugin.
- [ ] Update `android/app/src/main/AndroidManifest.xml` to include:
    - `<uses-permission android:name="android.permission.INTERNET" />` (Existing)
    - `<uses-permission android:name="android.permission.VIBRATE" />`
    - `<uses-permission android:name="android.permission.POST_NOTIFICATIONS" />` (Android 13+)

### 3. Capacitor Plugin Installation
Execute the following in the root directory:
```bash
npm install @capacitor/push-notifications
npx cap sync android
```

### 4. Application Logic Integration
- **Service Creation**: Implement `src/services/pushNotificationService.ts`.
- **Initialization**: Trigger `PushNotifications.requestPermissions()` on app mount or login.
- **Listeners**:
    - `registration`: Store the FCM token in the Supabase `profiles` table for targeted notifications.
    - `pushNotificationReceived`: Handle foreground notification displays.
    - `pushNotificationActionPerformed`: Handle deep-linking when a user taps a notification (e.g., opening a specific Booking).
