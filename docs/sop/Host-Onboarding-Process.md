# Host Onboarding Process

## Overview

This Standard Operating Procedure (SOP) defines the streamlined 8-step pipeline for onboarding private hosts onto the MobiRides platform. This process replaces legacy, manual onboarding workflows to focus aggressively on a platform-first, self-contained journey that maximizes conversion velocity while retaining our high standards for trust, validation, and fleet quality. 

---

## The 8-Step Onboarding Pipeline

### 1. Initial Call
**Objective:** Qualify the prospect, build rapport, and set expectations.
- Identify the prospect’s goals (e.g., passive income generation vs covering car payments).
- Discuss high-level platform economics (15% commission, host earnings structure).
- Outline the timeline and critical requirements for listing a vehicle.
- Answer immediate questions and overcome initial objections.

### 2. Send Onboarding Guide
**Objective:** Provide the host with the necessary async resources to formalize their understanding.
- Send the standardized Host Onboarding Guide PDF via Email/WhatsApp.
- Ensure the guide highlights the Pay-U Damage Protection to reassure the host of asset safety.
- Include a direct, trackable link to the platform signup page.

### 3. Platform Signup (Account Creation)
**Objective:** Get the host registered on the platform.
- Instruct the host to create an account on the MobiRides platform.
- The host provides basic contact information, sets up credentials, and successfully logs into the web or mobile app.

### 4. Host Validation
**Objective:** Confirm identity to uphold platform trust and safety before vehicles are made publicly available.
- Instruct the host to complete the in-app KYC flow (Identity Verification).
- Commercial team monitors admin dashboard for the pending verification request.
- Admin verifies and formally approves the host identity in the backend system.

### 5. Vehicle Listing (Fleet Data Collection)
**Objective:** Bring the host’s inventory into the database.
- The host follows the in-app prompts to add their vehicle.
- Host uploads required documentation (registration, current mileage).
- Host categorizes the vehicle and completes the 8-angle photography process using the app's upload functionality.
- The host establishes their dynamic or standard pricing baselines and availability calendar.

### 6. Customer Onboarding Support
**Objective:** Ensure the host is fully prepared to execute a successful handover and understand platform functions.
- **Tutorial Completion:** Ensure the host completes the mandatory in-app Tutorial Module, which educates them on the 7-step handover process, wallet payouts, and dispute resolution.
- Commercial rep conducts a brief touch-point (via WhatsApp or Call) to confirm the host feels confident with the app interface.

### 7. Vehicle Inspection Scheduling
**Objective:** Coordinate the physical quality-control step.
- Commercial rep reaches out to the host to book a mutually convenient time for the formal physical vehicle inspection.
- Confirm the inspection location (host premises or a central MobiRides hub).

### 8. Wallet Crediting
**Objective:** Facilitate the final administrative credits required to finalize the onboarding process.
- **Inspection Fee:** Deduct or appropriately allocate the agreed inspection fee within the host's wallet ledger.
- **Roadside Assistance Trial:** Credit the platform wallet to grant the host access to the complimentary Roadside Assistance trial package.
- Formally set the host's listing to 'Live/Active' for renter bookings.
