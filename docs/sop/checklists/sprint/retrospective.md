# Sprint Retrospective Checklist

Use this checklist for sprint retrospective meetings.

## Before Retrospective

### Data Gathering
- [ ] Collect sprint velocity
- [ ] Pull completed issues
- [ ] Review blockers raised
- [ ] Note anyadir comments

### Metrics Review
- [ ] Points completed vs committed
- [ ] Bug count introduced
- [ ] PR review time
- [ ] Deployment frequency

## During Retrospective

### Discussion Topics
- [ ] What went well?
- [ ] What didn't go well?
- [ ] What can we improve?
- [ ] Action items identified

### Team Feedback
- [ ] Positive highlights
- [ ] Pain points
- [ ] Suggestions

## After Retrospective

### Action Items
- [ ] Create action issues
- [ ] Assign owners
- [ ] Set due dates
- [ ] Add to next sprint
