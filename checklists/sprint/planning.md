# Sprint Planning Checklist

Use this checklist when preparing for sprint planning sessions.

## Before Planning

### Preparation
- [ ] Review previous sprint velocity
- [ ] Check team capacity
- [ ] Review backlog prioritization
- [ ] Prepare feature requests

### Requirements
- [ ] Draft user stories
- [ ] Define acceptance criteria
- [ ] Identify dependencies
- [ ] Estimate story points

## During Sprint Planning

### Discussion Points
- [ ] Review selected issues
- [ ] Clarify requirements
- [ ] Break down large tasks
- [ ] Identify blockers
- [ ] Assign owners

### Commitment
- [ ] Team commits to work
- [ ] Sprint goal defined
- [ ] Velocity cap respected
- [ ] Buffer for emergencies

## After Planning

### Actions
- [ ] Create Linear issues
- [ ] Assign to team members
- [ ] Set sprint dates
- [ ] Update project board
- [ ] Share with stakeholders
